# Technical Design: claude-md-sdd-injection

## Document Info
- Feature: claude-md-sdd-injection
- Status: Draft
- Created: 2026-03-13
- Requirements: ./requirements.md

## Architecture Overview

```
konbini init / update
        │
        ▼
┌─────────────────┐     ┌──────────────────────┐
│   init.ts        │────▶│  claude-md-injector.ts │
│   update.ts      │     │  (new module)          │
└─────────────────┘     └──────┬───────────────┘
                               │
                    ┌──────────┼──────────┐
                    ▼          ▼          ▼
            ┌───────────┐ ┌────────┐ ┌────────────┐
            │ templates/ │ │ao.yaml │ │ CLAUDE.md  │
            │ claude-md/ │ │(config)│ │ (project)  │
            │  en.md     │ └────────┘ └────────────┘
            │  ja.md     │
            └───────────┘
```

新モジュール `claude-md-injector.ts` が全ロジックを担う。`init.ts` と `update.ts` から呼び出す。テンプレートは `templates/claude-md/{lang}.md` に言語別ファイルとして配置。

**Rationale:** 既存の `template-copier.ts` はファイルコピーに特化しており、マーカーベースのセクション注入は責務が異なる。新モジュールに分離することで `template-copier.ts` の変更を最小限にし、テストも独立して書ける。

## Component Design

### Component: claude-md-injector

- **Purpose**: CLAUDE.md へのSDD強制ルール注入・更新を行う
- **Location**: `src/generators/claude-md-injector.ts`
- **Dependencies**: `fs`, `path`, `src/utils/paths.ts`, `src/utils/logger.ts`
- **Interface**:

```typescript
export interface ClaudeMdConfig {
  language: string;  // "en" | "ja"
  path: string;      // relative to project root, e.g. "CLAUDE.md"
}

// init 用: インタラクティブに設定を取得して注入
export async function injectClaudeMdInteractive(
  projectRoot: string
): Promise<ClaudeMdConfig>;

// update 用: ao.yaml の設定を使って注入
export function injectClaudeMd(
  projectRoot: string,
  config: ClaudeMdConfig
): void;

// テスト用に公開
export function detectLanguage(content: string): string;
export function buildSection(templateContent: string, version: string): string;
export function injectSection(existingContent: string, section: string): string;
```

### Component: Language Templates

- **Purpose**: 言語別のSDD強制ルールテンプレートを提供
- **Location**: `templates/claude-md/en.md`, `templates/claude-md/ja.md`
- **Dependencies**: なし
- **Interface**: Mustache-style `{{VERSION}}` プレースホルダを含むMarkdownファイル

### Component: ao.yaml Config Extension

- **Purpose**: `claude_md.language` と `claude_md.path` を ao.yaml に永続化
- **Location**: 既存の `templates/config/ao.yaml.template` に追記 + `src/utils/yaml.ts` を拡張
- **Dependencies**: `src/utils/yaml.ts`

### Component: init.ts / update.ts Changes

- **Purpose**: 既存フローに CLAUDE.md 注入ステップを追加
- **Location**: `src/commands/init.ts`, `src/commands/update.ts`
- **Dependencies**: `claude-md-injector.ts`

## API Contracts

### `injectClaudeMdInteractive(projectRoot)`

- **Signature**: `async (projectRoot: string) => Promise<ClaudeMdConfig>`
- **Input**: プロジェクトルートの絶対パス
- **Output**: ユーザーが選択した設定（language, path）
- **Behavior**:
  1. 既存 CLAUDE.md の存在確認、内容読み取り
  2. 言語自動検出（既存コンテンツから）またはデフォルト `en`
  3. インタラクティブプロンプト: 言語選択（検出結果をデフォルト表示）
  4. インタラクティブプロンプト: CLAUDE.md パス確認（デフォルト `CLAUDE.md`）
  5. テンプレート読み込み → バージョン置換 → セクション注入
  6. 設定を返す（呼び出し元が ao.yaml に保存）
- **Errors**: ファイル書き込み失敗時は `log.error()` で通知、例外は投げない

### `injectClaudeMd(projectRoot, config)`

- **Signature**: `(projectRoot: string, config: ClaudeMdConfig) => void`
- **Input**: プロジェクトルート + ao.yaml から読んだ設定
- **Output**: なし（CLAUDE.md を直接更新）
- **Errors**: ファイル書き込み失敗時は `log.error()` で通知、例外は投げない

### `detectLanguage(content)`

- **Signature**: `(content: string) => string`
- **Input**: CLAUDE.md の既存コンテンツ（マーカー外の部分）
- **Output**: `"ja"` | `"en"`
- **Logic**: 日本語文字（ひらがな・カタカナ・漢字）の存在率で判定。閾値以上なら `"ja"`、それ以外は `"en"`。
- **Rationale**: 複雑なNLP不要。CJK文字の有無で十分判定可能。サポート言語が en/ja の2言語なので、日本語かそれ以外かの二値判定で足りる。

### `buildSection(templateContent, version)`

- **Signature**: `(templateContent: string, version: string) => string`
- **Input**: テンプレートファイルの内容、konbini バージョン文字列
- **Output**: マーカー付きの完成セクション
- **Format**:
  ```
  <!-- konbini:sdd:start -->
  <!-- Generated by konbini v{version} — DO NOT EDIT THIS SECTION MANUALLY -->
  {templateContent with {{VERSION}} replaced}
  <!-- konbini:sdd:end -->
  ```

### `injectSection(existingContent, section)`

- **Signature**: `(existingContent: string, section: string) => string`
- **Input**: 既存 CLAUDE.md の全内容（空文字列可）、注入するセクション
- **Output**: セクション注入後の全内容
- **Logic**:
  1. 両マーカーが存在 → マーカー間を置換（マーカー含む）
  2. start のみ or end のみ存在 → 壊れた状態。既存のマーカー行はそのまま残し、末尾に新セクションを追加
  3. マーカーなし → 末尾に改行 + セクションを追加
  4. 既存コンテンツが空 → セクションのみ返す

## Data Models

### ao.yaml Extension

```yaml
# 既存の ao.yaml テンプレートに追加:
claude_md:
  language: {{CLAUDE_MD_LANG}}
  path: {{CLAUDE_MD_PATH}}
```

- `language`: `"en"` | `"ja"` — 注入セクションの言語
- `path`: CLAUDE.md の相対パス（プロジェクトルートから）

### Template File Format

```markdown
## SDD Enforcement Rules (CRITICAL)

<!-- konbini v{{VERSION}} -->

When you receive a request for ...
```

`{{VERSION}}` のみがプレースホルダ。テンプレート自体は純粋なMarkdown。

## State Management

```
init フロー:
  1. 既存 CLAUDE.md 読み取り (or empty)
  2. 言語検出
  3. ユーザー入力 (language, path)
  4. テンプレート読み込み + VERSION 置換
  5. セクション注入 → CLAUDE.md 書き込み
  6. config を返す → init.ts が ao.yaml に反映

update フロー:
  1. ao.yaml から claude_md 設定読み取り (or defaults)
  2. テンプレート読み込み + VERSION 置換
  3. 既存 CLAUDE.md 読み取り
  4. セクション注入 → CLAUDE.md 書き込み
```

## Error Handling Strategy

| Error | Recovery | User Message |
|-------|----------|-------------|
| CLAUDE.md 書き込み不可 | スキップして続行 | `log.error("CLAUDE.md の書き込みに失敗しました: {error}")` |
| テンプレートファイル不在 | スキップして続行 | `log.error("CLAUDE.md テンプレートが見つかりません: {lang}")` |
| ao.yaml に claude_md 設定なし (update時) | デフォルト値使用 | `log.info("claude_md 設定が見つかりません。デフォルトを使用します")` |
| 不正な言語指定 | `en` にフォールバック | `log.warn("未対応の言語: {lang}。English を使用します")` |

**Rationale:** CLAUDE.md 注入はクリティカルな機能ではないため、失敗してもinit/updateの他の処理を中断しない。

## Testing Strategy

### Unit Tests

- **`detectLanguage`**: 日本語テキスト → `"ja"`, 英語テキスト → `"en"`, 混在テキスト → 多い方, 空文字 → `"en"`
- **`buildSection`**: テンプレート + バージョン → マーカー付きセクション、`{{VERSION}}` が置換されること
- **`injectSection`**:
  - 空コンテンツ → セクションのみ
  - マーカーなしの既存コンテンツ → 末尾に追加
  - 両マーカーあり → 置換
  - 片方のマーカーのみ → 末尾に追加（壊れた状態のハンドリング）
  - マーカー間が空 → 注入
  - 既存コンテンツの前後の空白保持
- **`injectClaudeMd`**: ファイルI/Oの統合テスト（一時ディレクトリ使用）

### Integration Tests

- `konbini init` → CLAUDE.md が生成されること
- `konbini init` × 2 → 冪等であること
- `konbini update` → セクションのみ更新、ユーザーコンテンツ保持
- 既存 CLAUDE.md + `konbini init` → ユーザーコンテンツ保持

### Edge Cases

- CLAUDE.md が読み取り専用
- CLAUDE.md パスに存在しないディレクトリを指定
- ao.yaml が存在しない状態での update
- 巨大な CLAUDE.md（パフォーマンス問題なし確認）

## Security Considerations

- テンプレートファイルはパッケージにバンドルされるため、サプライチェーン攻撃のリスクは npm パッケージ自体と同等
- CLAUDE.md パスにディレクトリトラバーサル（`../../`）が指定された場合の影響 → パスはプロジェクトルートからの相対パスとして `path.resolve()` で解決するため、プロジェクトルート外への書き込みは可能だが、ユーザーが明示的に指定した場合のみ。CLI ツールとして妥当な範囲。
